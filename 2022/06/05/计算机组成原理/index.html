计算机组成原理

发表于2022-06-05|更新于2022-06-06|笔记
|阅读量:

第一章

计算机的衡量指标

机器字长：CPU一次能处理的二进制数据的位数。

决定着寄存器、运算部件、数据总线的位数。

机器字长越长，表示的数的范围越大，精度也越高，计算机字长有8、16、32、64位不等。

主频：主时钟产生时钟脉冲的固定频率。

主频越高，CPU的工作速度越快

运算速度：一般采用单位时间内执行指令的平均条数或浮点运算次数作为运算速度指标。

每秒执行百万条命令（MIPS）

每秒完成百万次浮点运算次数（MFPOPS）

CPI：执行一条指令所需要的时钟周期数。

平均无故障时间

第二章

数值数据的机内表示方法

进位计数制

按照一定的进位方法进行计数的数制称为进位计数制，简称进制。

通常在进行加法运算时逢几进一就称几进制。

进制间的相互转换

二进制、八进制和十六进制转换为十进制

按权展开转换法

十进制转换为二进制

（1）十进制整数转换成二进制整数

          减权定位法和除二取余法

（2）十进制小数转换成二进制小数

          减权定位法和乘二取整法

二进制转换成八进制、十六进制

（1）二进制转换成八进制

          从小数点的位置开始，整数部分向左、小数部分向右，每三位二进制数分为一组，对应一位八进制数，不足三位者补0，整数在高位补，小数在末位补。

（2）二进制转换成十六进制

          方法与二进制转换成八进制相同，每四位二进制数对应一位十六进制数。

八进制、十六进制转换成二进制

（1）八进制转换成二进制

          每一位八进制数用对应的三位二进制数代替，其整数中的最高位、小数部分的末位0可以省去不写。

（2）十六进制转换成二进制

          每一位十六进制数用对应的四位二进制数代替。

数的符号表示

有符号数和无符号数

机器数：将一个数连同符号一起数字化后在计算机中用二进制表示称为机器数。

无符号数：机器数的所有二进制位都用来表示数值，称为无符号数。

有符号数：将数的符号也数字化的数称为有符号数。通常用0表示‘+’号，1表示‘-’号。符号放在二进制数的最高位，称为符号位。

有符号数的表示

原码

最高位为符号位，其余为为数的绝对值。

补码

正数的补码与原码相同，都等于真值。

负数的补码将符号位用1表示，真值的数值位直接取反再在最低位加1。

补码的表示范围：（机器字长为n+1位）

小数：-1 <= X <= 1-2^(-n)

整数：-2^n <= X <= 2^n-1

反码

一般用作原码求补码或补码求原码的工具。

正数的反码与原码相同。

负数的反码将符号位用1表示，数值为按位取反。

三种码的比较

1.原码适合乘除运算，补码适合加减运算，在补码运算过程中符号位和数值为一起参加运算，只要不超过数的表示范围，结果都是正确的。

2.原码和反码所能表示的数是关于0对称的，整数+-(2^n - 1),小数+-(1-2^-n).

3.原码和反码有+- 0 ，而补码中的0的表示是唯一的.

数的小数点表示

定点表示法

数据的小数点的位置固定不变，

小数的小数点通常放在有效数字的前面，符号的后面；

整数的小数点就放在有效数字的末尾。

浮点表示法

将任意一实数采用科学计数法表示，小数点的位置不固定

在浮点表示法中，E称为浮点数的阶数，用定点整数表示，一般采用补码或移码表示；

M称为浮点数的尾数，用定点小数表示。

规格化浮点数

对于非规格化的浮点数，总是可以通过尾数移位和调整阶数的方法将其转化为规格化浮点数

规格化的浮点数表示是唯一的。

当基是2时，规格化的浮点数的尾数为+-0.1xxxx·····小数点后第一位必须是1

移码

如果阶码有n+1位，则阶数X的移码为

X移 = 2^n + X , -2^n <= X <= 2^n-1

十进制数据的表示

十进制数在计算机中是以十进制的二进制编码 BCD码来表示的。

十进制数据有权码

每个十进制数位的4个二进制位都有一个确定的权

常见的有8421码

十进制数据无权码

每个十进制数位的4个二进制位没有一个确定的权

常见的有格雷码（又称循环码）

格雷码的编码规则是任何两个相邻的代码只有一个二进制位不同，其余三位全部相同。

非数值数据的表示

汉字编码

汉字输入码

1.数字编码

2.字音编码

3.字形编码

4.行音编码

汉字交换码（国标码）

汉字交换码是在两个汉字处理系统之间进行汉字信息交换时使用的编码。

国标码 = 区位码 + 2020H

汉字机内码

汉字机内码时汉字信息处理系统在计算机内部存储和处理汉字信息时使用的编码。

机内码 = 国标码 + 8080H = 区位码 + A0A0H

第三章

定点数的加减法运算

补码加减法所依据的关系：

加法：（X + Y）补 = X补 + Y补

减法：（X - Y）补 = X补 + （-Y）补

（-Y）补是将 Y补 连同符号位一起取反后，再在最低位加1

补码加减法运算规则：

（1）参加运算的操作数用补码表示

（2）符号位等同于数值一起参加运算

（3）对于两数相加减的各种情况，计算机都执行加法操作

（4）结果用补码表示

溢出及检测方法

概念

计算机执行算数运算所产生的结果超出机器数所能表示的数的范围称为溢出

只有符号相同的两数进行加法运算才有可能产生溢出

检测方法

1）采用一位符号位判断

设A和B是两个操作数，S为运算结果

A = anan-1···a0

B = bnbn-1···b0

S = A + B =snsn-1···s0

若an=1,bn=1而sn=0,表示两个负数相加结果是整数，产生负溢出

若an=0,bn=0而sn=1,表示两个正数相加结果是负数，产生正溢出

2）用Cn-1和Cn判断

Cn-1表示最高数值位向符号位的进位

Cn表示符号本身产生的进位

在进行补码加法运算时，如果最高数值位和符号位不同时产生进位，则结果产生溢出

3）采用双符号位(变形补码)判断

00表示正数，11表示负数

两个正数（00）相加，结果符号位为01，表示有正溢出

两个负数（11）相加，结果符号位为10，表示有负溢出

二进制加法器

半加器

两个一位二进制数Ai,Bi相加（不考虑低位的进位），称为半加。

全加器

考虑低位的进位加法运算就是全加运算。

加法器

串行加法器

如果n位字长的加法器仅有一位全加器，使用移位寄存器从低位到高位串行提供操作数，分n步进行相加，称为串行加法器。

并行加法器

全加器个数和操作数位数相同，能同时对所有位求和。

1）串行进位加法器

将n个全加器的进位位按照串行串联方式，从低到高逐位依次连接起来。

2）先行进位加法器
